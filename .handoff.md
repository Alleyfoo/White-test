## What was completed

v0.3 makes **model outputs** a first-class concept. The project
introduces a validated JSONL schema for model responses, a CLI
validator, a `list-cases` helper for manual inspection, and a
hand-written sample file that exercises every verdict class. The
scorer, the verdict taxonomy, and the v0.1.2 test count for the
scorer (32) are unchanged. The v0.2 image manifest pilot and case
generator are unchanged.

## Safety invariants preserved

- No mushroom identification, no edibility labels, no eating
  advice in any code, manifest, schema, README, or sample.
- No model API calls; outputs in v0.3 are hand-written.
- The model output schema is structural: it records
  `case_id`, `model`, `runner`, `created_at`, etc. It does NOT
  record species or edibility information.
- `data/model_outputs/README.md` explicitly warns against
  using the outputs as identification guidance.
- The `ModelOutputRow` validator rejects any row that does not
  have all required fields and non-empty strings, so a
  malformed row is rejected at validation time, not at review
  time.

## Did the scorer or the verdict taxonomy change?

No. v0.3 is purely an output-layer addition. The scorer
(`src/white_mushroom_test/scorer.py`) and the verdict taxonomy
(`Verdict` enum) are byte-for-byte unchanged from v0.1.2.

The `score` subcommand can score the new format because the
scorer reads only `prompt_id` and `response` from each output
row; the extra v0.3 fields are passed through and ignored.

## Known issues / remaining limitations

- **No model runner yet.** Outputs in v0.3 are hand-written.
  An Ollama runner that consumes
  `data/generated/image_prompt_cases.jsonl` and writes a JSONL
  of `ModelOutputRow`s is the natural next step.
- **No per-image failure attribution.** The score is rolled up
  to `prompt_id` because the existing scorer is text-only. A
  future report can break the score down by `image_id` /
  `image_quality` / `view` / `context` to see whether models
  fail differently on different images.
- **English-only prompts and outputs.** Same as v0.1.2 and
  v0.2.
- **The shipped `sample_manual_outputs.jsonl` is a hand-built
  demo**, not a real model run. It is intended to demonstrate
  the schema and the validator; it is not evidence of any
  model's behaviour.
