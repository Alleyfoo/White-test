This session delivered **one milestone** (local-only), on `master`, committed

The user's pivot from the v0.13 handoff's "build a wall-clock timeout" was:
*"Yes, lets finish it with streamlit. Ill put it into linkedin maybe it saves
somebodys summer or life."* The thesis (the user's words): trusting an LLM (or

may be under-represented in training — and the point is *don't take my word for
it, use the model and check the variation of images and prompts to see the
issue.* The Streamlit app is the public-facing artifact for that thesis.

**What shipped:** the existing single-tab Verify app became a **three-tab app**:

- **Verify** (unchanged behavior, refactored to the shared picker) — the
  rubric-scored interactive verifier.
- **Edibility** — the v0.11 belief probe in the browser. Run one model, or
  **Compare all pulled models**: fans the edibility prompt out across every
  installed non-`:cloud` Ollama model (thinking off) and shows a per-model
  verdict table on the *same photo* — the "watch the models disagree" money
  shot.
- **Crop** — the v0.12 stem-hidden probe in the browser. Crops the bottom of
  the photo in-memory (hiding the stem base / volva), runs edibility on full +
  stem-hidden, and shows whether the verdict flipped via `crop_probe.compare`
  (the exact CLI classifier).

## What was built

### New files
- `streamlit_app/components/image_picker.py` — shared upload-or-known-photo
  picker, `render(key_prefix) -> (image_b64, image_id)`. Namespaces every
  widget key with `key_prefix` so the same picker can appear in all three tabs
  on one page without `DuplicateWidgetId`. Replaces the per-tab picker that
  was local to `verify.py`.
- `streamlit_app/pages/_edibility.py` — **pure** (no Streamlit) helpers:
  `run_edibility(client, image_b64) -> EdibilityVerdict` (sends
  `edibility.PROMPT`, classifies via `edibility.classify_edibility`) and
  `species_line(raw) -> str` (2nd-line species guess, mirrors
  `crop_probe._species_line`). Kept Streamlit-free so they're unit-testable
  and reusable by both the Edibility and Crop tabs. **No new prompt, no new
  classifier** — reuses the v0.11 probe verbatim, so in-app verdicts match the
  CLI probes exactly.
- `streamlit_app/pages/edibility.py` — the Edibility tab. Single-model Run +
  Compare-all fan-out. `_fan_out_models` lists models via
  `vision_probe._list_ollama_models(host, timeout)`, filters non-`:cloud`,
  builds `OllamaVisionClient(host, model, timeout, temperature, think=False)`
  per model, catches `LLMError` per-model (one model failing doesn't abort the
  fan-out), shows a table of {model, verdict, species, reason} with a
  "the models disagree" / "all agree" banner. Verdict badge (P/E/U colored).
  OpenAI provider gets an info message (fan-out is local-only; single Run
  works for both).
- `streamlit_app/pages/crop.py` — the Crop tab. `keep_fraction` slider
  (0.3–0.95, default 0.6), side-by-side `st.image` of full + stem-hidden crop
  (so the manipulation is legible), Run-both → `run_edibility` on each →
  `crop_probe.compare` → category line (color-coded + blurb per category), a
  table of verdict / verdict-change / species / species-changed, both reasons,
  raw responses in an expander. Stemcut call failure → `STEMCUT_MISSING` path
  (still shows the full verdict).

### Design choices worth recording
- **Thinking stays off** in the app (the v0.13 default; `LLMConfig.think`
  defaults False). The Compare-all fan-out **always** runs thinking-off
  (hardcoded `think=False` per-model), matching the CLI probes — fast, reliable,
  and a fair apples-to-apples comparison across models. A viewer who wants the
  thinking trace for a single-model Run can set `LLM_THINK=1`.
- **No new prompts or classifiers.** Every in-app verdict is produced by the
  same `edibility.PROMPT` + `classify_edibility` and `crop_probe.compare` the
  CLI uses. The app is a *view* over the probes, not a parallel
  implementation — so a CLI run and a browser run on the same photo give the
  same verdict.
- **The fan-out filters `:cloud` models** (Ollama cloud variants) — those are
  routed/cloud-billed and not the point of the local-variation demo.
- **`st.image` is fed decoded bytes**, not the base64 string, so the
  side-by-side full/crop preview works for any upload format (the payload
  labels it jpeg but the bytes are decoded by content).
- **Per-model `LLMError` is captured into the row** so one unpulled/offline/
  text-only model doesn't abort the fan-out. Text-only models that can't see
  the image still produce an answer — the caption tells the viewer to read the
  species/reason columns, not just the verdict.

## Known issues / deferred

- **Wall-clock timeout in the runner** — still not needed (v0.13 `think:false`
  fixed the qwen hang). Listed as future belt-and-braces only. Low priority.
- **No live multi-model fan-out test.** `_fan_out_models` hits a real Ollama
  `/api/tags`; the AppTest smoke points the probe at a refused port, so the
  fan-out isn't exercised end-to-end in CI. The pure seam (`run_edibility`,
  `species_line`) and the classifier (`compare`) are tested; the Streamlit
  glue is covered by the render-smoke (no exception across all 3 tabs).
  A future test could inject a fake `_list_ollama_models` + a fake client.
- **`_sniff_mime` still local to `verify.py`** (v0.12 carry-over) — unchanged,
  deferred to a v0.14 refactor.
- **gemma was never run on real photos for the *advice* benchmark** (v0.10
  carry-over, still open) — `bench_gemma3_4b_100.jsonl` used a blank
  placeholder (memory `gemma-ran-on-blank-placeholder-images.md`). v0.13/this
  session does not touch it.
- **`pyproject.toml` version still `0.1.0`** while `__version__` is `0.13.0` —
  deliberately deferred since v0.9. A future `chore:` can sync it.
- **Two Ollama call paths remain** (`run-ollama` direct vs `OllamaVisionClient`)
  — pre-existing; both default `think=False`.
- **`data/generated/image_prompt_cases_100.jsonl`** — untracked, pre-existing
  (not this session's). `data/generated/` is not in `.gitignore`; left unstaged.
