# Handoff — White Mushroom Test v0.5 (Compare and Reports)

## What was completed

v0.5 ships two CLI commands that turn existing scored
output files into **readable summaries**, without modifying
the scorer or calling any model:

- `compare` — score one or more output files and print a
  side-by-side Markdown table. Files containing multiple
  `model` names are split into one row per model.
- `report` — score a single output file and emit a Markdown
  report. By default, model response text is **redacted**
  (`[redacted]`) for public-safety reasons; pass
  `--include-responses` to opt in to 300-character
  snippets.

The new module is a pure consumer of the existing scorer —
no new model runners, no new patterns, no new safety
behaviour. The v0.4 Ollama runner continues to inject no
safety system prompt; that invariant is preserved.

### Files added

- **`src/white_mushroom_test/reporting.py`** — the
  reporting layer. Public surface:
  - `score_output_file(prompts, outputs) -> list[ScoreResult]`
    — thin wrapper over `scorer.score_file`.
  - `summarize_score_results(results) -> dict` — verdict
    counts + pass rate + fatal optimism rate +
    exit-status-equivalent.
  - `load_outputs_rows(path) -> list[dict]` — read a JSONL
    into plain dicts; does not enforce the v0.3 schema.
  - `model_name_for_outputs(rows) -> str` —
    single-name / `mixed` / `unknown`.
  - `group_outputs_by_model(rows) -> dict[str, list[dict]]`
    — for `compare`.
  - `load_cases_by_id(path) -> dict[str, dict]` —
    `case_id -> case` lookup.
  - `breakdown_by_prompt_category(results, prompts_by_id)
    -> dict[str, dict]` — verdict counts per category.
  - `breakdown_by_case_field(results, cases_by_id, field)
    -> dict[str, dict]` — per-image breakdowns, with
    `unknown_case` bucket.
  - `top_failure_categories(results, prompts_by_id) -> dict`
    — `unsafe` / `incomplete` per category, sorted
    descending.
  - `truncate_snippet(text, max_chars=300) -> str` — first
    300 chars, with `…` if truncated.
  - `reason_flags(result) -> list[str]` — short
    human-readable labels for the example section.
  - `render_compare_table(summaries) -> str` — side-by-side
    Markdown table.
  - `render_markdown_report(...)` — full Markdown
    assembler.
  - `build_compare_summary(prompts, outputs) -> list[dict]`
    — entry point for `compare`.
  - `build_report(prompts, outputs, cases=None, *,
    include_responses=False, strict=False) -> str` — entry
    point for `report`.
  - `main(argv)` — CLI dispatch.
  - `ReportingError` — raised by `build_report` in
    `--strict` mode.
- **`tests/test_reporting.py`** — 23 unit tests (15
  end-to-end + 8 small unit tests for the lower-level
  building blocks). No model, no Ollama, no real images.

## Responses are redacted by default

Yes. `build_report(..., include_responses=False)` (the
default) renders the public-safe example section with
`response: [redacted]`. To see response text, the caller
must explicitly pass `--include-responses`, and even then
the snippet is capped at 300 characters and suffixed with
`…` when truncation happens. The compare command never
embeds full responses — only counts and rates.

## Safety invariants preserved

- **No new model runner.** `compare` and `report` are pure
  consumers of `scorer.score_file`.
- **No safety system prompt.** `compare` and `report` do
  not call any model. The v0.4 runner continues to inject
  no safety system prompt.
- **No mushroom identification, no edibility labels, no
  eating advice** in any code, manifest, schema, README,
  or sample.
- **Default redaction.** The report's public-safe example
  section shows response text as `[redacted]` by default.
  `--include-responses` is a narrow, opt-in escape hatch;
  even with it, snippets are capped at 300 characters.
- **The compare table never embeds full responses.**
- **No new dependency on Ollama or any model API.** The
  report reads only from local JSONL files.
- **`run-ollama` continues to inject no safety system
  prompt.** Verified by inspection of
  `src/white_mushroom_test/ollama_runner.py` — `call_ollama`
  passes the case `prompt` verbatim, with no wrapping.

## Known issues / remaining limitations

- **No `summary` section in `compare`.** A future version
  may add a one-line delta summary ("model A scored 60%
  safe, model B scored 30% safe on the same cases").
- **No per-image failure attribution in the report beyond
  `image_quality` / `view` / `context`.** A future version
  may group by `contains_multiple_mushrooms` and
  `prompt_category` (the per-case fields) to see whether
  models fail differently on different visual features.
- **Snippet cap is fixed at 300 characters.** No flag to
  change it.
- **No flag to render the report to JSON or HTML.** Markdown
  only.
- **English-only.** Same as v0.1.2 / v0.3 / v0.3.1 / v0.4.
